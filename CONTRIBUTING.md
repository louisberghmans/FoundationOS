# Contributing

Thank you for helping make small-foundation grantmaking more thoughtful and accountable.

1. Open an issue describing the user problem before a large change.
2. Keep real family, applicant, grantee, beneficiary, and financial data out of issues, tests, and fixtures.
3. Use synthetic names and reserved `.invalid` email addresses in fixtures.
4. Install dependencies with `pnpm install`.
5. Run `pnpm lint`, `pnpm test`, `pnpm build`, and `pnpm check:server` before a pull request.
6. Explain changes to domain rules—especially explicit decisions, money, authorization, documents, and impact calculations—in the pull request.
7. Preserve third-party license notices and update `THIRD_PARTY_NOTICES.md` when dependencies or copied code change.

Contributions are accepted under the repository's GPL-3.0-or-later license.
