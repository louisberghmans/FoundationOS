# Contributing

Thank you for helping make family philanthropy more thoughtful and accountable.

1. Open an issue describing the user problem before a large change.
2. Keep real family, grantee, and beneficiary data out of issues, tests, and fixtures.
3. Install dependencies with `pnpm install`.
4. Run `pnpm lint`, `pnpm test`, and `pnpm build` before submitting a pull request.
5. Explain changes to domain rules—especially voting, money, authorization, and outcome calculations—in the pull request.

The project uses TypeScript, React, and conventional commits where practical.
